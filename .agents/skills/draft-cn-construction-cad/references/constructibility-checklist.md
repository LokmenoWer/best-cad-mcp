# 施工可用性校核清单

交付 DWG/PDF 前执行本清单。发现问题先修图，再保存或导出。

## 基础校核

- 模型空间按 1:1 绘制，单位为毫米；视口/布局控制打印比例。
- 图框、标题栏、图名、图号、比例、日期、版本、专业、项目名称齐全。
- 字体可用，中文不乱码，PDF 导出后文字仍可读。
- 所有正式实体位于中文业务图层，颜色/线型/线宽主要为 `bylayer`。
- `0` 层、`Defpoints`、临时层、英文占位层中没有正式构件。
- 辅助线、不打印线、重复线、孤立短线、测试文字已清理。
- 闭合边界真的闭合；填充使用 hatch 对象且边界可追溯。
- 尺寸是 dimension 对象，标注值与几何实测一致。
- 块命名清楚，重复构件用块/阵列，块基点合理。
- 线型比例、文字比例、填充比例在目标打印比例下可读。

## 专业校核

- 建筑：轴号连续，墙厚、门窗洞口、楼梯、房间名称/面积、标高、剖切和详图索引完整。
- 结构：构件编号、截面、标高、配筋/构造索引、节点详图、施工说明一致。
- 给排水：系统、管径、坡度、标高、阀件、洁具/设备编号、套管或预留洞清楚。
- 暖通：风管/水管尺寸、标高、流向、阀件、设备编号、防火阀和保温说明清楚。
- 电气：回路编号、箱柜编号、灯具/插座/弱电点位、桥架/管线、系统图索引清楚。
- 消防：喷头、报警点、消火栓、阀门、管径、防火分区或系统关系清楚。
- 总图：红线、坐标、建筑定位、道路、竖向标高、室外管线和现状参照清楚。

## 查询建议

使用仓库提供的查询工具做机器校核：

```sql
SELECT layer, COUNT(*) AS n
FROM cad_entities
GROUP BY layer
ORDER BY n DESC;
```

```sql
SELECT handle, type, layer
FROM cad_entities
WHERE layer IN ('0', 'Defpoints') OR layer LIKE 'Layer%';
```

```sql
SELECT type, COUNT(*) AS n
FROM cad_entities
GROUP BY type
ORDER BY n DESC;
```

重点检查是否存在大量孤立 `AcDbLine` 组成的表格、矩形、剖面线或尺寸。若有，改用 `draw_rectangle`、`add_table`、`add_hatch`、`add_*_dimension` 等高级对象。
